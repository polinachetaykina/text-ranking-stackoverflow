# Text Ranking for StackOverflow Questions

Поиск похожих вопросов на StackOverflow с использованием классических и нейросетевых методов ранжирования. Проект сравнивает BM25, TF-IDF, Word2Vec и SBERT для задачи semantic search.

Проект является исследовательской работой по сравнению методов ранжирования текстов.

## Исследовательская часть

### Датасет

| Параметр | Значение |
|----------|----------|
| Train samples | 1,000,000 пар вопросов |
| Validation samples | 3,760 пар |
| Источник | StackOverflow duplicate questions |

### Предобработка данных

- Очистка от HTML-тегов и специальных символов
- Приведение к нижнему регистру
- Удаление пунктуации
- Токенизация (NLTK)
- Удаление стоп-слов
- Лемматизация

### Сравниваемые методы

| Метод | Тип | Особенность |
|-------|-----|-------------|
| BM25 | Классический | Учёт длины документа, насыщение TF |
| TF-IDF | Классический | Униграммы + биграммы |
| Word2Vec | Классический | Усреднение эмбеддингов (200d) |
| SBERT | Нейросетевой | Sentence-BERT fine-tuning |

### Результаты обучения

| Метод | MRR | Hits@1 | Hits@5 | Hits@10 |
|-------|-----|--------|--------|---------|
| BM25 | 0.609 | 0.526 | 0.696 | 0.756 |
| TF-IDF | 0.513 | 0.412 | 0.612 | 0.688 |
| Word2Vec | 0.507 | 0.424 | 0.602 | 0.676 |
| SBERT | 0.760 | 0.690 | 0.840 | 0.880 |

### Вывод исследования

BM25 был выбран как лучший классический метод благодаря:
- Лучшей метрике MRR (0.609)
- Учёту длины документа
- Отсутствию необходимости в обучении

SBERT показал результаты лучше, чем все классические методы (0,76)

## Jupyter Notebook

### Функциональность

- Загрузка и предобработка данных
- Обучение классических методов (BM25, TF-IDF, Word2Vec)
- Дообучение SBERT с MultipleNegativesRankingLoss
- Визуализация результатов (графики Hits@K, DCG@K, MRR)

### Метрики оценки

| Метрика | Формула | Описание |
|---------|---------|----------|
| MRR | 1/N * Σ(1/rank) | Среднее обратных рангов |
| Hits@K | #{rank ≤ K} / N | Доля ответов в топ-K |
| DCG@K | Σ 1/log₂(rank+1) | Учитывает и релевантность, и позицию документа |
